# Demo GIF Recording Checklist

This is the **only manual step** in the documentation pipeline.
All other figures are generated by `make figures`.

## Target file

`docs/assets/demo.gif` — 10 to 15 seconds, looping.

## Tool

Recommended: [LICEcap](https://www.cockos.com/licecap/) (cross-platform, free) or
[Kap](https://getkap.co/) (macOS). Record as GIF directly.

Alternative: record MP4 with OBS/ffmpeg then convert:
```bash
ffmpeg -i demo.mp4 -vf "fps=12,scale=900:-1:flags=lanczos" -loop 0 docs/assets/demo.gif
```

## Setup before recording

1. Start the backend: `uv run uvicorn apps.api.main:app --reload`
2. Start the frontend: `cd apps/web && npm run dev`
3. Open `http://localhost:5173` in a browser at **900 x 600 px** viewport.
4. Use image: `ml/data/DsPCBSD+/Data_YOLO/images/val/0014089.jpg` (has multiple defect classes).

## What to show (in order)

1. The empty drop zone — 1 s pause so the viewer can register the UI.
2. Drag and drop the PCB image into the drop zone.
3. The bounding boxes appearing on the canvas with class labels.
4. Scroll / click to the diagnostic panel — the repair sheet filling in.
5. End on the filled repair sheet — 1 s pause.

## Resolution and quality

- Viewport: 900 x 600 px minimum.
- Frame rate: 10-15 fps (lower = smaller file).
- Target file size: < 5 MB.

## After recording

Replace this file with the actual GIF:
```bash
mv /path/to/demo.gif docs/assets/demo.gif
git add docs/assets/demo.gif
git commit -m "docs(assets): add demo GIF"
```
