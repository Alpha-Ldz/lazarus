# Anomaly Detection Benchmark — PatchCore on VisA PCB

## Motivation

YOLOv11s and RT-DETR-l are **supervised** detectors: they require thousands of box-annotated
images per class. In real industrial settings, this annotation cost is the blocking factor.

PatchCore answers the question the current benchmark does not pose:
**what if I had zero defect annotations?**

It trains only on healthy PCB images (a memory bank of features, no backpropagation) and flags
any deviation from normal. This ticket adds the **annotation cost** axis to the decision matrix —
a business decision, not a leaderboard contest.

## Dataset

**VisA** — Visual Anomaly dataset, categories `pcb1`, `pcb2`, `pcb3`, `pcb4`.

- Train split: normal images only (no defect annotations required).
- Test split: normal + defective images with pixel-level ground truth masks.
- Auto-downloaded by Anomalib on first run.

## Model

**PatchCore** (Roth et al., 2022) with Anomalib default backbone.

- No backpropagation: the "training" phase builds a coreset memory bank from patch embeddings.
- Inference: patches of the test image are compared to the memory bank; distance drives the anomaly score.
- Fit time: order of minutes per category (not hours).

## Metrics

| Metric | Description |
|--------|-------------|
| **AUROC Image** | ROC-AUC at image level (normal vs anomalous) |
| **AUROC Pixel** | ROC-AUC at pixel level (localization quality) |
| **AU-PRO** | Area Under the Per-Region Overlap curve — rewards localizing the right region even if size is off |

## Non-Comparability with the Supervised Benchmark

> **These results are NOT comparable** to the mAP/P/R metrics in `docs/BENCHMARK.md`.

| Axis | Supervised (YOLO / RT-DETR) | Unsupervised (PatchCore) |
|------|-----------------------------|-----------------------------|
| **Dataset** | DsPCBSD+ (9 classes, box annotations) | VisA PCB (normal/anomalous, no boxes) |
| **Metrics** | mAP50, mAP50-95, Precision, Recall | AUROC image, AUROC pixel, AU-PRO |
| **Resolution** | 640×640 | 256×256 (Anomalib default) |
| **Supervision** | Thousands of annotated boxes per class | Normal images only (0 defect annotations) |
| **Latency** | Not comparable (different resolutions) | Not comparable (different resolutions) |

**No ratio between mAP and AUROC makes sense.** Do not attempt to aggregate these columns.

### What IS legitimately comparable

- **Annotation cost**: YOLO/RT-DETR require thousands of defect boxes annotated per class;
  PatchCore only needs healthy images — near-zero annotation cost.
- **Training cost**: PatchCore (memory bank, no backpropagation) fits in minutes per category
  vs hours for supervised models.
- **Localization capability**: qualitatively, via exported heat maps.
  PatchCore bounding boxes are approximations (connected components of the thresholded mask).

### Threshold strategy

Anomalib computes an adaptive threshold (`F1AdaptiveThreshold`) during fitting.
This threshold is stored on the model and used at inference time.

Fallback (if not available): **percentile 99 of the anomaly map**, documented as such in
`apps/api/detectors/patchcore.py`.

## Swappability Proof

PatchCore plugs into the production API without modifying `analyze.py` or `main.py`:

```bash
LAZARUS_DETECTOR=patchcore \
LAZARUS_MODEL_PATH=ml/runs/patchcore/pcb1/weights/torch/model.pt \
LAZARUS_PATCHCORE_CATEGORY=pcb1 \
uv run uvicorn apps.api.main:app
```

The `/api/analyze` endpoint returns `class_name: "anomaly"` detections.
The `/api/analyze/classes` endpoint returns `["anomaly"]`.
The `analyze.py` router remains untouched — the `Detector` abstraction holds.

## Known Limitations

- **`diagnose.txt` prompt** expects a `defect_type` from the 9 DsPCBSD+ classes. With PatchCore
  it receives `"anomaly"` — the LLM produces a generic repair sheet. Not a blocking bug;
  next logical ticket is to adapt the diagnosis prompt for the anomaly paradigm.
- **Bounding box quality**: PatchCore is a pixel-level anomaly detector. The bbox extraction
  (connected components of the thresholded heat map) is a pragmatic approximation, not a true
  object detector.
- **Resolution mismatch**: the 256×256 Anomalib default is not tuned for PCB images.
  Higher resolution may improve pixel-AUROC at the cost of inference speed.
